    《PowerShell 新手高频实用指令整理》

    对于计算机科学与技术专业的大一新生来说，Windows PowerShell 是重要的系统管理和自动化工具，但不存在 “专业最常用的 50 条指令” 这一绝对标准—— 不同场景（如文件管理、系统监控、进程控制、网络排查）下常用指令差异极大，且新手无需一次性掌握大量指令，优先掌握高频核心指令更高效。

    以下按「新手必备基础」「文件 / 目录管理」「系统监控与控制」「网络排查」「用户与权限」「进阶自动化」6 大场景，整理了40 + 条高频实用指令，每条均标注「功能说明」和「新手示例」，帮你快速上手：

    一、新手必备基础指令（5 条）

    这是使用 PowerShell 的 “入门钥匙”，先掌握这些再学其他指令：

    指令 功能说明 新手示例
    Get-Help 查看任何指令的帮助文档（核心！不会就查它） Get-Help Get-ChildItem（查文件列表指令用法）
    Get-Command 搜索系统中所有可用的指令 Get-Command *file*（搜索包含 “file” 的指令）
    cd（Set-Location） 切换当前工作目录（和 CMD 一致，简化写法） cd D:\专业课资料（切换到 D 盘的 “专业课资料” 文件夹）
    cls（Clear-Host） 清空当前 PowerShell 窗口内容 输入cls回车，窗口内容清空
    exit 关闭当前 PowerShell 窗口 输入exit回车，直接退出

    二、文件与目录管理（12 条）

    专业学习中频繁需要管理代码文件、实验数据，这组指令是核心：

    指令 功能说明 新手示例
    Get-ChildItem（ls/dir） 列出当前目录下的文件 / 子目录（ls是简化写法，和 Linux 一致） ls（列当前目录内容）、ls -a（显示隐藏文件）
    New-Item 创建新文件或新目录 New-Item 实验1代码.py（创建 Python 文件）、New-Item 实验报告 -ItemType Directory（创建 “实验报告” 文件夹）
    Copy-Item（cp） 复制文件 / 目录 cp 实验1代码.py D:\备份（把文件复制到 D 盘备份文件夹）
    Move-Item（mv） 移动文件 / 目录（也可用于重命名） mv 实验1代码.py 实验1_最终版.py（重命名文件）、mv 实验1_最终版.py D:\课程作业（移动文件）
    Remove-Item（rm） 删除文件 / 目录（谨慎使用！删除后难恢复） rm 废弃文件.txt（删除文件）、rm 旧实验文件夹 -Recurse（删除带子目录的文件夹）
    Rename-Item（ren） 专门用于重命名文件 / 目录 ren 课程笔记.md 计算机组成原理笔记.md（重命名文档）
    Get-Content（cat） 查看文本文件内容（如代码、日志） cat 实验1代码.py（查看 Python 文件的代码内容）
    Set-Content 覆盖文本文件内容（慎用，会清空原有内容） Set-Content 测试.txt "Hello PowerShell"（创建或覆盖测试文件内容）
    Add-Content 向文本文件末尾追加内容（不覆盖原有内容） Add-Content 测试.txt "——来自CS新生"（在文件末尾加一行内容）
    Test-Path 检查文件 / 目录是否存在（脚本中常用） Test-Path D:\专业课资料（判断这个文件夹是否存在，返回 True/False）
    Measure-Object 统计文件内容（如行数、字符数，适合代码统计） `Get-Content 实验 1 代码.py
    Compress-Archive 压缩文件 / 目录为 ZIP 包（备份代码常用） Compress-Archive D:\课程作业 作业备份.zip（把 “课程作业” 文件夹压缩成 ZIP）

    三、系统监控与进程控制（8 条）

    调试程序时需查看进程、CPU / 内存占用，这组指令帮你排查 “程序卡死”“资源占用过高” 问题：

    指令 功能说明 新手示例
    Get-Process（ps） 查看当前运行的所有进程（如 Python、VS Code） ps（列所有进程）、ps -Name python（只看 Python 相关进程）
    Stop-Process（kill） 强制结束进程（程序卡死时用） kill -Name python（结束所有 Python 进程）、kill -Id 1234（通过进程 ID 精准结束某进程，ID 从ps结果中看）
    Get-Service 查看系统服务（如数据库服务、远程服务） Get-Service mysql（查看 MySQL 数据库服务是否在运行）
    Start-Service/Stop-Service 启动 / 停止系统服务 Start-Service mysql（启动 MySQL 服务）、Stop-Service mysql（停止 MySQL 服务）
    Get-Counter 查看系统资源占用（CPU、内存、磁盘 IO） Get-Counter "\Processor(_Total)\% Processor Time"（查看 CPU 总占用率）
    Get-Disk 查看磁盘分区信息（容量、剩余空间） Get-Disk（查看所有磁盘的基本信息）、Get-Volume（更直观查看分区容量和剩余空间）
    Get-EventLog 查看系统日志（排查系统错误、程序崩溃原因） Get-EventLog -LogName Application -Newest 10（查看应用程序日志的最新 10 条记录）
    systeminfo 查看系统详细信息（如操作系统版本、硬件配置） 直接输入systeminfo，会显示电脑的 CPU、内存、系统版本等所有信息

    四、网络排查（7 条）

    做网络编程、远程连接服务器时，需排查网络通断、IP 配置，这组指令是 “网络诊断工具”：

    指令 功能说明 新手示例
    ping 测试目标 IP / 域名是否能连通（基础网络排查） ping baidu.com（测试能否连百度，判断网络是否正常）、ping 192.168.1.1（测试能否连路由器）
    tracert 追踪数据包到目标的路径（排查哪里断网） tracert baidu.com（查看数据从你电脑到百度经过的所有节点）
    Get-NetIPAddress 查看电脑的 IP 地址、子网掩码等网络配置 Get-NetIPAddress -AddressFamily IPv4（只看 IPv4 地址，更常用）
    Get-NetAdapter 查看网卡信息（如 WiFi、有线网卡是否启用） Get-NetAdapter（查看所有网卡状态，判断 WiFi 是否正常）
    Test-Connection PowerShell 版 “ping”，功能更灵活 Test-Connection baidu.com -Count 3（只 ping 3 次，比默认ping更省时间）
    netstat 查看网络连接状态（如端口占用，调试网络程序用） netstat -ano（查看所有端口的占用情况，-o显示对应进程 ID）
    Resolve-DnsName 解析域名对应的 IP 地址（DNS 排查） Resolve-DnsName baidu.com（查看百度域名对应的 IP）

    五、用户与权限（5 条）

    后续做服务器管理、多用户环境时会用到，大一先了解基础：

    指令 功能说明 新手示例
    Get-LocalUser 查看当前电脑的所有本地用户 Get-LocalUser（显示电脑上的所有用户账号）
    New-LocalUser 创建新的本地用户（需管理员权限） New-LocalUser "CS_Student" -NoPassword（创建一个叫 “CS_Student” 的无密码用户）
    Get-LocalGroup 查看本地用户组（如管理员组、普通用户组） Get-LocalGroup（显示所有用户组）
    Get-Acl 查看文件 / 目录的权限（谁能读 / 写 / 修改） Get-Acl D:\专业课资料（查看这个文件夹的权限设置）
    Set-Acl 修改文件 / 目录的权限（需谨慎，避免权限混乱） 新手暂不推荐操作，后续学权限管理再深入

    六、进阶自动化基础（4 条）

    PowerShell 的核心优势是 “自动化”，大一可先了解基础脚本指令，为后续编程自动化打基础：

    指令 功能说明 新手示例
    ForEach-Object（%） 遍历集合（如批量处理文件、数据） `ls *.txt
    Where-Object（?） 筛选数据（如筛选大文件、特定进程） `ls
    Export-Csv 将数据导出为 CSV 文件（方便用 Excel 分析） `Get-Process
    .\_脚本名.ps1 运行 PowerShell 脚本文件（自动化的核心） 先创建批量备份.ps1脚本（写好复制文件的指令），再输入.\批量备份.ps1运行脚本

    新手学习建议：

    1. 不要死记硬背：先掌握 “场景对应指令”（如要复制文件就想到cp），用Get-Help 指令名查细节。

    2. 从 “模仿” 到 “实践”：先跟着示例敲指令，再尝试用指令完成实际需求（如备份代码、统计代码行数）。

    3. 注意权限：部分指令（如修改系统服务、创建用户）需 “以管理员身份打开 PowerShell”（右键 PowerShell 选择 “以管理员身份运行”）。

    随着后续学习（如操作系统、网络编程、自动化脚本），你会自然掌握更多指令，现阶段优先用好上述核心指令即可。
